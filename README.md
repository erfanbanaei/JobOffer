# JobOffer Bot

ربات تلگرامی که آگهی‌های استخدامی جدید رو از چند سایت کاریابی (جابینجا، جاب‌ویژن، ای‌استخدام، ایران‌تلنت) هر ۱۵ دقیقه چک می‌کنه و برای کاربرها می‌فرسته. هر کاربر با یه ویزارد دکمه‌ای (کلمه‌ی جستجو → سایت‌ها → شهر → نوع همکاری) می‌تونه هم‌زمان روی چندتا سایت سرچ بسازه.

## معماری

- **Django** — مدل‌ها (`accounts.TelegramUser`, `jobs.SearchQuery`, `jobs.JobPosting`)، پنل ادمین، و ORM
- **aiogram 3** — پردازش پیام‌ها و دکمه‌های تلگرام (فقط `/start` و `/help` به‌عنوان دستور؛ بقیه از طریق دکمه) با long polling
- **Celery + Celery Beat** — هر ۱۵ دقیقه صفحه‌ی سرچ هر کاربر رو می‌گیره، آگهی‌های جدید رو پارس می‌کنه و پیام تلگرام می‌فرسته
- **Redis** — بروکر Celery
- **PostgreSQL** — دیتابیس اصلی
- **requests + BeautifulSoup** — اسکرپ صفحه‌ی نتایج جابینجا (سایت API/RSS رسمی ندارد)

پردازش‌ها در ۴ پروسه‌ی جدا اجرا می‌شن: `web` (پنل ادمین)، `bot` (پولینگ تلگرام)، `worker` (اجرای تسک‌های اسکرپ)، `beat` (زمان‌بند هر ۱۵ دقیقه).

## راه‌اندازی

1. از [@BotFather](https://t.me/BotFather) یک بات بساز و توکنش رو بگیر.
2. فایل `.env` رو از روی نمونه بساز:
   ```bash
   cp .env.example .env
   ```
   و مقدار `TELEGRAM_BOT_TOKEN` رو با توکن واقعی پر کن. مقادیر دیگه (Postgres/Redis) رو می‌تونی همون پیش‌فرض بگذاری.

3. **پراکسی (لازم از ایران):** چون `api.telegram.org` در ایران مسدوده، باید `TELEGRAM_PROXY_URL` رو هم در `.env` تنظیم کنی. اگه از V2Ray/Xray استفاده می‌کنی، پورت SOCKS5 محلی‌ش رو پیدا کن (در v2rayN/v2rayNG معمولاً «Local SOCKS Listening Port»، پیش‌فرض اغلب `10808` یا `1080`) و بگذار:
   ```
   TELEGRAM_PROXY_URL=socks5://host.docker.internal:10808
   ```
   نکته: داخل کانتینر داکر باید `host.docker.internal` بنویسی نه `127.0.0.1` (چون پراکسی روی خود سیستم تو اجرا می‌شه، نه داخل کانتینر). اگه بدون Docker و مستقیم روی سیستم خودت اجرا می‌کنی، باید `127.0.0.1` بگذاری.

4. با Docker بالا بیار:
   ```bash
   docker compose up --build
   ```
   این کار Postgres، Redis، پنل ادمین (پورت 8000)، بات، Celery worker و Celery beat رو همزمان اجرا می‌کنه.

5. برای ساخت کاربر ادمین (اختیاری، برای دیدن داده‌ها در `/admin`):
   ```bash
   docker compose exec web python manage.py createsuperuser
   ```

## بدون Docker (اجرای محلی)

```bash
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # و POSTGRES_HOST / REDIS_URL رو به localhost تغییر بده
python manage.py migrate
python manage.py runserver         # پنل ادمین
python manage.py runbot            # در یک ترمینال جدا: بات تلگرام
celery -A jobbot worker -l info    # در ترمینال جدا
celery -A jobbot beat -l info      # در ترمینال جدا
```

## استفاده در تلگرام

**از این به بعد فقط دو دستور وجود داره: `/start` و `/help`.** بقیه‌ی عملکردها فقط از طریق دکمه‌ها در دسترسن (نه دستور تایپی) — هم در کیبورد ثابت پایین صفحه، هم دکمه‌های شیشه‌ای داخل پیام‌ها.

بعد از `/start` یه کیبورد دائمی با ۵ دکمه نمایش داده می‌شه:

- **➕ افزودن سرچ** — یه ویزارد ۴ مرحله‌ای: اول **کلمه‌ی جستجو** رو می‌پرسه (مثلاً flutter)، بعد **سایت‌ها** رو با چک‌باکس چندانتخابی نشون می‌ده (مثلاً هم‌زمان جابینجا و جاب‌ویژن رو تیک بزنی، با یه سرچ هم‌زمان روی هر دو ساخته می‌شه)، بعد شهر/استان (یا «🌍 همه‌ی ایران»)، و در آخر نوع همکاری (تمام‌وقت/پاره‌وقت/کارآموزی/دورکاری، چندانتخابی). با تأیید نهایی، به ازای هر سایت انتخاب‌شده یه سرچ جدا ساخته می‌شه.
- **📋 سرچ‌های من** — لیست سرچ‌ها با دکمه‌های شیشه‌ای رنگی زیر هر کدوم: 🔵 توقف موقت / 🟢 ازسرگیری، و 🔴 حذف. در پایین لیست هم یه دکمه‌ی «➕ افزودن سرچ جدید» هست.
- **👤 حساب کاربری** — نام، نام خانوادگی، یوزرنیم، آیدی عددی و تعداد سرچ‌های فعالت رو نشون می‌ده.
- **🆘 پشتیبانی** — یه دکمه‌ی شیشه‌ای برای ارتباط مستقیم با ادمین (`@MrTakDev`) برای پیشنهاد یا گزارش مشکل.
- **❓ راهنما** — توضیح کامل نحوه‌ی کار ربات.

> **نکته درباره‌ی «رنگ دکمه»:** API بات تلگرام امکان تغییر رنگ واقعی پس‌زمینه‌ی دکمه‌های شیشه‌ای رو نمی‌ده (این گزینه در Bot API وجود ندارد). به‌جاش از ایموجی دایره‌ی رنگی (🔴 قرمز برای حذف، 🟢 سبز برای مشاهده/ازسرگیری، 🔵 آبی برای توقف) به‌عنوان نزدیک‌ترین معادل بصری استفاده شده.

**عضویت اجباری در کانال:** قبل از هر تعامل با ربات (حتی `/start`)، عضویت در کانال `@erfanbanaei_ir` چک می‌شه. اگه عضو نباشه، پیام عضویت با دکمه‌ی لینک کانال و دکمه‌ی «✅ عضو شدم، بررسی کن» نشون داده می‌شه. **این یعنی بات باید بتونه عضویت رو از طریق Bot API چک کنه** — برای کانال‌های عمومی معمولاً بدون نیاز به ادمین‌بودن بات کار می‌کنه، ولی اگه با خطا مواجه شدی، بات رو به‌عنوان ادمین کانال اضافه کن تا `getChatMember` همیشه درست جواب بده. این منطق در [bot/middlewares.py](bot/middlewares.py) پیاده‌سازی شده.

وقتی یه سرچ تازه اضافه می‌شه، فقط **آخرین آگهی** برات ارسال می‌شه (نه همه‌ی نتایج موجود) — بقیه به‌عنوان خط پایه ذخیره می‌شن تا فقط آگهی‌های واقعاً جدید در اسکن‌های بعدی برات بیاد. هر آگهی با یه دکمه‌ی شیشه‌ای «🟢 مشاهده و ارسال رزومه» می‌آد، نه لینک خام.

## منابع آگهی (Providers)

هر منبع (جابینجا، جاب‌ویژن، ای‌استخدام، ایران‌تلنت) یه فایل جدا داره در [jobs/providers/](jobs/providers/) با:

- `LABEL`, `DOMAIN` — برای نمایش در ویزارد و تشخیص خودکار توسط دیسپچر اسکرپ
- `build_url(keyword, city, job_types)` — لینک قابل‌نمایش بر اساس فیلترهای انتخابی می‌سازه.
- `fetch_jobs(url)` — همون لینک رو می‌خونه (یا API می‌زنه) و لیست آگهی‌ها رو برمی‌گردونه.

ویژگی‌های مشترک (لیست استان‌ها، گزینه‌های نوع همکاری) در [jobs/providers/common.py](jobs/providers/common.py) هستن تا ویزارد بات مستقل از منبع کار کنه. هر چهار منبع فعلی روش متفاوتی دارن:

- **جابینجا** ([jobinja.py](jobs/providers/jobinja.py)) — HTML سرور-رندرشده، با BeautifulSoup پارس می‌شه.
- **جاب‌ویژن** ([jobvision.py](jobs/providers/jobvision.py)) — اپ Angular با رندر سمت کلاینت؛ صفحه‌ی نتایج آگهی نداره، آگهی‌ها از API واقعی‌ش (`candidateapi.jobvision.ir/api/v1/JobPost/List`) به‌صورت JSON گرفته می‌شن.
- **ای‌استخدام** ([eestekhdam.py](jobs/providers/eestekhdam.py)) — اپ Nuxt با رندر سمت کلاینت؛ از API داخلی‌ش (`www.e-estekhdam.com/search-api/search`) استفاده می‌شه. نکته: همون اندپوینت `slug-params` که خودِ سایت برای پارس کردن اسلاگ صفحه به فیلتر استفاده می‌کنه رو صدا می‌زنیم، به‌جای اینکه خودمون اسلاگ رو پارس کنیم — مطمئن‌تره (مخصوصاً برای اسم استان‌های چندکلمه‌ای).
- **ایران‌تلنت** ([irantalent.py](jobs/providers/irantalent.py)) — اپ Angular با رندر سمت کلاینت، اما برخلاف جاب‌ویژن هیچ API JSON قابل‌کشفی از باندل JS پیدا نشد؛ در عوض، سایت برای کراولرهای SEO (Googlebot و مشابه) نسخه‌ی کامل سرور-رندرشده برمی‌گردونه که شامل یه بلاک `<script id="ng-state">` با کل دیتای جستجو (Angular Universal TransferState) می‌شه. اسکرپر با یه User-Agent شبیه Googlebot درخواست می‌زنه و این بلاک JSON رو مستقیماً پارس می‌کنه — این یه تکنیک رایج و قانونی SEO («دیناميک رندرینگ») است، نه دورزدن کنترل دسترسی. نکته: ترتیب پیش‌فرض نتایج این سایت بر اساس relevance (نه تاریخ) است، پس اسکرپر قبل از برگرداندن لیست، خودش بر اساس `created_at` نزولی مرتب می‌کنه تا منطق «فقط جدیدترین» در [jobs/tasks.py](jobs/tasks.py) درست کار کنه.

برای هر منبع جدید اول باید بررسی شه که HTML سرور-رندرشده (مثل جابینجا) دارد یا باید API واقعی‌ش پیدا شه (با گرفتن باندل JS سایت و grep کردن دنبال اندپوینت‌های API، یا اگه پیدا نشد، تست کردن رفتار سایت با User-Agent بات‌های SEO مثل Googlebot).

برای اضافه کردن یه منبع جدید: یه فایل provider مشابه بساز، `LABEL`/`DOMAIN` و `build_url`/`fetch_jobs` رو پیاده‌سازی کن، و در [jobs/providers/__init__.py](jobs/providers/__init__.py) به دیکشنری `PROVIDERS` اضافه‌ش کن — ویزارد بات و دیسپچر اسکرپ خودکار می‌شناسنش.

## محدودیت‌های فعلی

- جابینجا، جاب‌ویژن، ای‌استخدام و ایران‌تلنت پشتیبانی می‌شن.
- فقط صفحه‌ی اول نتایج (جدیدترین‌ها) چک می‌شه، نه تمام صفحات.
- ای‌استخدام تاریخ دقیق انتشار آگهی رو در پاسخش نمی‌ده؛ فقط یک پرچم «جدید» دارد، پس فیلد زمان انتشار برای این منبع ضعیف‌تره.
- ایران‌تلنت فقط استان‌هایی که فعلاً آگهی فعال دارن رو در لیست شهرهاش برمی‌گردونه؛ ۳ استان (ایلام، خراسان شمالی، کهگیلویه و بویراحمد) فعلاً در لیست داخلی [irantalent.py](jobs/providers/irantalent.py) نیستن و انتخابشون فقط فیلتر شهر رو نادیده می‌گیره (بدون خطا).
- اگه ساختار HTML/API این سایت‌ها تغییر کنه، پارسر/کلاینت مربوطه در `jobs/providers/` باید آپدیت شه. روش‌های scrape مبتنی بر کشف غیررسمی (باندل JS یا رفتار SEO) شکننده‌تر از API رسمی هستن.
