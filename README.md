# JobOffer Bot

ربات تلگرامی که آگهی‌های استخدامی جدید جابینجا رو هر ۱۵ دقیقه چک می‌کنه و برای کاربرها می‌فرسته. هر کاربر می‌تونه چندتا لینک سرچ جابینجا (مثل سرچ flutter در تهران) رو با دستور `/addsearch` اضافه کنه.

## معماری

- **Django** — مدل‌ها (`accounts.TelegramUser`, `jobs.SearchQuery`, `jobs.JobPosting`)، پنل ادمین، و ORM
- **aiogram 3** — پردازش دستورات تلگرام (`/start`, `/addsearch`, `/mysearches`, `/removesearch`) با long polling
- **Celery + Celery Beat** — هر ۱۵ دقیقه صفحه‌ی سرچ هر کاربر رو می‌گیره، آگهی‌های جدید رو پارس می‌کنه و پیام تلگرام می‌فرسته
- **Redis** — بروکر Celery
- **PostgreSQL** — دیتابیس اصلی
- **requests + BeautifulSoup** — اسکرپ صفحه‌ی نتایج جابینجا (سایت API/RSS رسمی ندارد)

پردازش‌ها در ۴ پروسه‌ی جدا اجرا می‌شن: `web` (پنل ادمین)، `bot` (پولینگ تلگرام)، `worker` (اجرای تسک‌های اسکرپ)، `beat` (زمان‌بند هر ۱۵ دقیقه).

## راه‌اندازی

1. از [@BotFather](https://t.me/BotFather) یک بات بساز و توکنش رو بگیر.
2. فایل `.env` رو از روی نمونه بساز:
   ```bash
   cp .env.example .env
   ```
   و مقدار `TELEGRAM_BOT_TOKEN` رو با توکن واقعی پر کن. مقادیر دیگه (Postgres/Redis) رو می‌تونی همون پیش‌فرض بگذاری.

3. **پراکسی (لازم از ایران):** چون `api.telegram.org` در ایران مسدوده، باید `TELEGRAM_PROXY_URL` رو هم در `.env` تنظیم کنی. اگه از V2Ray/Xray استفاده می‌کنی، پورت SOCKS5 محلی‌ش رو پیدا کن (در v2rayN/v2rayNG معمولاً «Local SOCKS Listening Port»، پیش‌فرض اغلب `10808` یا `1080`) و بگذار:
   ```
   TELEGRAM_PROXY_URL=socks5://host.docker.internal:10808
   ```
   نکته: داخل کانتینر داکر باید `host.docker.internal` بنویسی نه `127.0.0.1` (چون پراکسی روی خود سیستم تو اجرا می‌شه، نه داخل کانتینر). اگه بدون Docker و مستقیم روی سیستم خودت اجرا می‌کنی، باید `127.0.0.1` بگذاری.

4. با Docker بالا بیار:
   ```bash
   docker compose up --build
   ```
   این کار Postgres، Redis، پنل ادمین (پورت 8000)، بات، Celery worker و Celery beat رو همزمان اجرا می‌کنه.

5. برای ساخت کاربر ادمین (اختیاری، برای دیدن داده‌ها در `/admin`):
   ```bash
   docker compose exec web python manage.py createsuperuser
   ```

## بدون Docker (اجرای محلی)

```bash
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # و POSTGRES_HOST / REDIS_URL رو به localhost تغییر بده
python manage.py migrate
python manage.py runserver         # پنل ادمین
python manage.py runbot            # در یک ترمینال جدا: بات تلگرام
celery -A jobbot worker -l info    # در ترمینال جدا
celery -A jobbot beat -l info      # در ترمینال جدا
```

## استفاده در تلگرام

بعد از `/start` یه کیبورد دائمی با ۳ دکمه نمایش داده می‌شه:

- **➕ افزودن سرچ** — یه ویزارد ۴ مرحله‌ای: اول **منبع** رو می‌پرسه (جابینجا یا جاب‌ویژن)، بعد کلمه‌ی جستجو (مثلاً flutter)، بعد شهر/استان با دکمه‌های شیشه‌ای (یا گزینه‌ی «🌍 همه‌ی ایران» برای رد کردن فیلتر شهر)، و در آخر نوع همکاری (تمام‌وقت/پاره‌وقت/کارآموزی/دورکاری) که چندتاش رو هم‌زمان می‌تونی انتخاب کنی، با زدن «➡️ تأیید و ادامه» نهایی می‌شه. لینک از روی همین فیلترها خودکار ساخته می‌شه — کاربر هیچ لینکی دستی وارد نمی‌کنه.
- **📋 سرچ‌های من** — لیست سرچ‌ها رو با دکمه‌های شیشه‌ای (inline) زیر هر کدوم نشون می‌ده: یکی برای توقف/ازسرگیری موقت (⏸/▶️) و یکی برای حذف (🗑). با لمس دکمه، لیست همون‌جا آپدیت می‌شه.
- **❓ راهنما** — نمایش راهنما.

دستورات معادل: `/addsearch` همون ویزارد رو شروع می‌کنه، `/mysearches` همون لیست با دکمه‌های شیشه‌ای، `/removesearch <id>` حذف یک سرچ با آی‌دی.

وقتی یه سرچ تازه اضافه می‌شه، فقط **آخرین آگهی** برات ارسال می‌شه (نه همه‌ی نتایج موجود) — بقیه به‌عنوان خط پایه ذخیره می‌شن تا فقط آگهی‌های واقعاً جدید در اسکن‌های بعدی برات بیاد. هر آگهی با یه دکمه‌ی شیشه‌ای «🔗 مشاهده و ارسال رزومه» می‌آد، نه لینک خام.

## منابع آگهی (Providers)

هر منبع (جابینجا، جاب‌ویژن، ای‌استخدام، و بعداً ایران‌تلنت) یه فایل جدا داره در [jobs/providers/](jobs/providers/) با:

- `LABEL`, `DOMAIN` — برای نمایش در ویزارد و تشخیص خودکار توسط دیسپچر اسکرپ
- `build_url(keyword, city, job_types)` — لینک قابل‌نمایش بر اساس فیلترهای انتخابی می‌سازه.
- `fetch_jobs(url)` — همون لینک رو می‌خونه (یا API می‌زنه) و لیست آگهی‌ها رو برمی‌گردونه.

ویژگی‌های مشترک (لیست استان‌ها، گزینه‌های نوع همکاری) در [jobs/providers/common.py](jobs/providers/common.py) هستن تا ویزارد بات مستقل از منبع کار کنه. هر سه منبع فعلی روش متفاوتی دارن:

- **جابینجا** ([jobinja.py](jobs/providers/jobinja.py)) — HTML سرور-رندرشده، با BeautifulSoup پارس می‌شه.
- **جاب‌ویژن** ([jobvision.py](jobs/providers/jobvision.py)) — اپ Angular با رندر سمت کلاینت؛ صفحه‌ی نتایج آگهی نداره، آگهی‌ها از API واقعی‌ش (`candidateapi.jobvision.ir/api/v1/JobPost/List`) به‌صورت JSON گرفته می‌شن.
- **ای‌استخدام** ([eestekhdam.py](jobs/providers/eestekhdam.py)) — اپ Nuxt با رندر سمت کلاینت؛ از API داخلی‌ش (`www.e-estekhdam.com/search-api/search`) استفاده می‌شه. نکته: همون اندپوینت `slug-params` که خودِ سایت برای پارس کردن اسلاگ صفحه به فیلتر استفاده می‌کنه رو صدا می‌زنیم، به‌جای اینکه خودمون اسلاگ رو پارس کنیم — مطمئن‌تره (مخصوصاً برای اسم استان‌های چندکلمه‌ای).

برای هر منبع جدید اول باید بررسی شه که HTML سرور-رندرشده (مثل جابینجا) دارد یا باید API واقعی‌ش پیدا شه (مثل جاب‌ویژن/ای‌استخدام — با گرفتن باندل JS سایت و grep کردن دنبال اندپوینت‌های API).

برای اضافه کردن یه منبع جدید: یه فایل provider مشابه بساز، `LABEL`/`DOMAIN` و `build_url`/`fetch_jobs` رو پیاده‌سازی کن، و در [jobs/providers/__init__.py](jobs/providers/__init__.py) به دیکشنری `PROVIDERS` اضافه‌ش کن — ویزارد بات و دیسپچر اسکرپ خودکار می‌شناسنش.

## محدودیت‌های فعلی

- جابینجا، جاب‌ویژن و ای‌استخدام پشتیبانی می‌شن.
- فقط صفحه‌ی اول نتایج (جدیدترین‌ها) چک می‌شه، نه تمام صفحات.
- ای‌استخدام تاریخ دقیق انتشار آگهی رو در پاسخش نمی‌ده؛ فقط یک پرچم «جدید» دارد، پس فیلد زمان انتشار برای این منبع ضعیف‌تره.
- اگه ساختار HTML/API این سایت‌ها تغییر کنه، پارسر/کلاینت مربوطه در `jobs/providers/` باید آپدیت شه.
